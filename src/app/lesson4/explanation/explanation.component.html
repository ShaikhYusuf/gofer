Lesson 4: Writing and Loading a Gofer Script

Introduction

The Gofer interpreter operates interactively using the REPL (Read-Evaluate-Print Loop).

Even multi-line expressions are evaluated as single expressions in this environment.

Names, Values, and Bindings

Bindings in Gofer use the "=" symbol, and they are immutable:

  • height = 178
  • x = [1...4]

Example of local bindings:

  • let x = 10 in x + 2
  • x + y where (x, y) = (10, 20)

Gofer Scripts

A script is a plaintext file with extension .gs containing definitions.

Example script:

  p = 100
  q = 10.0
  r = [1...4]
  z = y + 3
  y = 2

Use the command :load lsn04.gs or :l lsn04.gs to load the script into the interpreter.

Interpreter Commands

Common commands:

  • :load or :l: Load a script
  • :reload or :r: Reload the last loaded script
  • :type or :t: Show the type of a variable or expression
  • :names or :n: List all known names
  • :?: Display help

Exercises

Try the following:

  1. Create a Gofer script with variables and bindings as shown above.
  2. Experiment with expressions like p + y or p ^ 2.
  3. Swap the order of bindings (e.g., define z = y + 3 before y = 2) and observe the behavior.
  4. Test proper and improper alignments to learn about indentation errors.
