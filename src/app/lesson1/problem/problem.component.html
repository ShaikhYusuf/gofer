Introduction to Functional Programming - Gofer

Problem 1

Enter the following expressions at the interpreter prompt, one at a time:

2
2.1
False
'a'
2+2
2.1 + 3.2
2<3
'2'
2*3
+ 4
True
3<2
' '

Notice that the interpreter prints out the type of each expression (e.g., 2 : Int, True : Bool).

Problem 2

Verify that the types of different expressions are printed out with a capital or upper-case letter (e.g., Int, Float, Char, Bool).

Problem 3

Enter expressions like 2^5 and 2.0^5 to verify that ^ is the exponentiation (power of) operator.

Problem 4

Enter the following and observe their behavior:

2+2 == 4
2+2 == 5
2 /= 3
2 /= 4
True == False
True /= False

Conclusion: == and /= are Boolean-valued equality and inequality operators that return True or False.

Problem 5

Enter expressions like 2+2 = 4 and verify the result.

Note that the "=" symbol means “equality by definition” in functional programming.

Problem 6

Try entering expressions like False&&False, False&&True, False||True, True||True and observe their values. These are the OR and AND operators for Boolean values.

Problem 7

Try entering incorrect expressions like 2 * 3 *, .1., and 'a' to see the error messages.

If you enter something like xyz, it will return “ERROR: Undefined variable.” If it starts with an uppercase letter, it will return “ERROR: Undefined constructor function.”
