Lesson 5: Introduction to Functional Programming

IPMF 2021

This lesson introduces the term-rewriting model of computation and its comparison with the state-change model. The focus is on declarative programming and equational reasoning using Gofer.

Theoretical Background

Equational reasoning is the basis of functional programming. It treats equality as a definition (e.g., 2 + 2 = 4) rather than an assignment action.

Story Example

The example of a student calculating simple interest showcases how Gofer uses term-rewriting, unlike C or Python, which follow a state-change model.

Classroom Exercise

Classroom Equation Gofer Script
SI = P * N * R / 100 si = p * n * (r / 100.0)
Amt = P + SI amt = p + si
P = Rs 2000 p = 2000.0
N = 3 years n = 3.0
R = 15% pa r = 15.0

Key Concepts

  • Term-Rewriting Model: Equations are facts, and values are derived through substitution.
  • State-Change Model: Variables are assigned values, causing memory state changes.
  • Declarative vs. Imperative: Declarative programming defines facts, while imperative programming gives instructions.
  • Referential Transparency: Function values depend only on inputs and produce the same result every time for the same input.

Comparison with Other Languages

In languages like C or Python, the equal sign (=) signifies assignment rather than equality, making them imperative programming languages.
